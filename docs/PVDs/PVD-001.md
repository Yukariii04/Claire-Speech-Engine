# Project Vision Document (PVD-001)

## Why CSE Exists

The Claire Speech Engine (CSE) exists to provide a production-grade, highly performant, and stable foundation for advanced AI speech synthesis. Most open-source speech research produces models with fragmented, research-grade code that is difficult to deploy into production environments or integrate seamlessly into real-time applications. CSE solves this by separating the **framework** from the **model**. 

By building a robust, agnostic framework first, we ensure that as speech models evolve, they can be deployed instantly into a stable, well-tested infrastructure.

## Project Philosophy

1. **Agnostic by Design**: CSE does not assume the internal architecture of any specific speech model. 
2. **Minimal and Lazy**: We favor deletion over addition. Features are only added when absolutely necessary. There is no bloat, no speculative abstractions, and no "just in case" architecture.
3. **Immutability and Thread Safety**: Audio frames and capabilities are immutable. The registry and runtime are thread-safe.

## Backend Independence

CSE treats speech generation backends as swappable plugins via the `AcousticBackend` interface. Whether using Kokoro, Fish Speech, StyleTTS2, or future state-of-the-art models, the public API remains completely identical. This allows developers to hot-swap models without changing their application logic, decoupling downstream applications from underlying ML model churn.

## Performance Before Acoustics

Acoustic quality is irrelevant if the engine cannot deliver audio in real-time. CSE was designed with strict performance constraints enforced by automated benchmarking:
- Initialization under 20ms
- Package import under 50ms
- Backend lookup under 10ms

We built the high-performance streaming architecture (`cse.streaming`) and the performance timeline compilation (`cse.performance`) *before* introducing real machine learning weights. This guarantees that the engine's overhead is virtually zero, allowing the ML models to utilize the entire hardware budget.

## Long-Term Roadmap

With the stabilization of CSE at **v1.0.0-beta**, the framework is feature-frozen. The project's future development splits into distinct layers:

1. **CSE (Claire Speech Engine)**: The core framework (maintenance and stability).
2. **CPE (Claire Performance Engine)**: The semantic intelligence layer (prosody, emotion, pacing, and dialog planning).
3. **CSM (Claire Speech Model)**: The actual acoustic machine learning models.

This separation of concerns allows us to iterate aggressively on speech intelligence without destabilizing the framework that developers depend on.
