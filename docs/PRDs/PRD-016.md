# 1. Executive Summary

Performance Context is the official input contract for the Claire Performance Engine (CPE).

Every request entering CPE SHALL be represented as a Performance Context.

Performance Context provides all information required for communication reasoning while remaining independent of backend implementation.

-------------------------------------------------------------------------------

# 2. Background

The Claire Performance Engine receives information from Claire Speech Engine before communication reasoning begins.

To ensure a stable architecture, every request SHALL use a standardized Performance Context.

This document defines that contract.

-------------------------------------------------------------------------------

# 3. Objectives

The Performance Context SHALL:

- Provide a stable input interface for CPE.
- Remain backend-independent.
- Support optional contextual information.
- Preserve compatibility with future extensions.
- Remain simple and deterministic.

-------------------------------------------------------------------------------

# 4. Scope

## In Scope

- Input contract
- Required fields
- Optional fields
- Validation requirements
- Compatibility requirements

## Out of Scope

- Communication reasoning
- Performance planning
- Backend translation
- Acoustic synthesis
- Runtime implementation

-------------------------------------------------------------------------------

# 5. Functional Requirements

### Required Field

Every Performance Context SHALL contain:

- Text

Text SHALL preserve the original user input.

---

### Optional Field

Performance Context MAY contain:

- Character State

When supplied, Character State SHALL be used by CPE as contextual information.

When absent, CPE SHALL infer the required performance from the supplied text.

-------------------------------------------------------------------------------

# 6. Validation Requirements

A valid Performance Context SHALL:

- Contain non-empty text.
- Preserve the original text.
- Accept Character State as optional.
- Reject malformed input.
- Produce deterministic validation results.
