# 1. Executive Summary

Implement the **Voice Runtime**.

The Voice Runtime is responsible for orchestrating the speech pipeline.

It receives a **Performance Timeline**, manages the active **Voice Package**, communicates with the **Acoustic Backend**, and returns synthesized audio.

**For this PRD, no real synthesis occurs.**

The backend will be a stub.

- Backend registration
- Public runtime API

- Acoustic inference

# Why this PRD exists

This PRD establishes the **control point** of the speech engine.

Later, whether the backend is:
- your own Claire acoustic model,
- a temporary backend,
- or another implementation,

the rest of the engine won't change.

The runtime becomes the stable interface between the planning side of CSE and the synthesis side.
