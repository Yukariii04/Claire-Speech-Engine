# 1. Executive Summary

Implement the **Audio Streaming Pipeline**.

This subsystem transports synthesized audio from the Acoustic Backend to future consumers (audio output, Unity, WebSocket, file writer, etc.).

It does **not** synthesize audio.

It does **not** play audio.

It only defines how audio is transported safely and efficiently.

### One note before you implement

This is the first PRD where I'd slightly tighten the implementation guidance compared to earlier ones:

- Keep the streaming API **PCM-format agnostic** (don't assume 16-bit integers; the frame metadata should describe the format).
- Keep the `AudioFrame` immutable.
- Make `StreamBuffer` generic enough that a future backend can stream very small chunks (low latency) or larger chunks (batch mode) without changing the API.

Those constraints will make the streaming layer usable whether your future backend is your own Claire acoustic model, Kokoro during development, or another backend you experiment with later.
