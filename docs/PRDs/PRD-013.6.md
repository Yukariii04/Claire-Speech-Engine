# 1. Executive Summary

PRD-013.5 successfully established the evaluation backend architecture and adapter interfaces for Fish Speech and StyleTTS2.

However, both adapters currently contain placeholder implementations that generate dummy WAV files rather than performing real speech synthesis.

The purpose of PRD-013.6 is to replace every placeholder implementation with the official upstream inference pipelines while preserving the existing Claire Speech Engine architecture.

No architectural redesign is permitted.

---

# 2. Objectives

Implement

- Official Fish Speech inference
- Official StyleTTS2 inference
- Real model loading
- Real tokenizer loading
- Real waveform generation
- Proper PCM WAV output
- Backend dependency validation
- Backend error reporting
- End-to-end Colab validation

Do NOT implement

- Claire backend
- Performance Engine
- Runtime redesign
- SpeechEngine redesign
- Public API changes
- New notebook

---

# 3. Existing Infrastructure Review (Mandatory)

Before writing any implementation code:

Inspect the existing Claire2D project.

Specifically review the existing Colab notebook and supporting utilities.

Reuse existing implementations wherever possible.

Review:

- Google Drive mounting
- Runtime initialization
- Dependency installation
- Reference audio handling
- Transcript handling
- Model download workflow

Do not redesign working infrastructure.

Reuse before rewrite.

---

# 4. Fish Speech Backend

Replace every placeholder implementation.

The backend SHALL:

- Load the official Fish Speech model.
- Load the required tokenizer.
- Initialize the official inference pipeline.
- Accept text through the existing backend interface.
- Generate real speech.
- Save valid PCM WAV output.
- Return a valid SpeechResult.

The backend SHALL NOT:

- Create placeholder WAV files.
- Return silent audio.
- Return zero-length WAVs.
- Raise NotImplementedError.
- Simulate inference.

---

# 5. StyleTTS2 Backend

Replace every placeholder implementation.

The backend SHALL:

- Load the official StyleTTS2 model.
- Initialize the official inference pipeline.
- Generate real speech.
- Save valid PCM WAV output.
- Return a valid SpeechResult.

The backend SHALL NOT:

- Generate placeholder audio.
- Simulate inference.
- Return empty WAVs.

---

# 6. Public API

The public API SHALL remain unchanged.

The following workflow SHALL continue to work.

```python
from cse import SpeechEngine

engine = SpeechEngine()

engine.load_backend("fishspeech")

engine.load_voice("default")

speech = engine.speak(
    "Hello from the Claire Speech Engine."
)
```

Only the backend name may change.

No additional public methods are permitted.

---

# 7. COLAB-001

The existing COLAB-001 notebook SHALL be upgraded.

Do NOT create another notebook.

COLAB-001 SHALL remain the official engineering and evaluation notebook.

It SHALL continue using the existing Claire2D workflow wherever appropriate.

The notebook SHALL:

- Mount Google Drive using the existing workflow.
- Install dependencies.
- Install CSE.
- Load Fish Speech.

# 8. Transcript Handling

Before changing transcript loading:

Inspect the Claire2D notebook.

# 9. Audio Validation

Generated WAV files SHALL:

- contain valid PCM samples
- have duration greater than zero
- play successfully
- contain audible synthesized speech

Returning only a WAV header SHALL be considered a failed implementation.

---

# 10. Dependency Handling

The backend SHALL:

- detect missing dependencies
- report actionable installation guidance
- fail gracefully

Unexpected stack traces for common setup issues should be avoided where practical.
