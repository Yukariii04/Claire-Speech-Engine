# 1. Executive Summary

Implement the **Performance Compiler**.

The Performance Compiler transforms a **Claire Intermediate Representation (CIR)** into a **Performance Timeline**.

It does **NOT** generate audio.

It decides **how** speech should be performed.

---

# 2. Purpose

The Performance Compiler is responsible for converting structured language into a deterministic performance plan.

It determines:

- speech pacing
- pauses
- hesitation
- emphasis
- breathing
- performance attributes

The Acoustic Backend will later synthesize audio from this timeline.

# 📌 Why PRD-003 is intentionally "simple"

You might notice this compiler isn't "smart" yet.

That's intentional.

We're separating **infrastructure** from **intelligence**.

Right now we're proving that the pipeline works:

```
CIR
   ↓
Performance Timeline
```

Later PRDs will incrementally add:

- pause prediction
- emphasis rules
- breathing rules
- pacing rules
- hesitation rules
- emotional progression
- context awareness

Because the architecture is already in place, those improvements won't require changing the public API.
