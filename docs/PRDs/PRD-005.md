# 1. Executive Summary

Implement the **Acoustic Backend Interface**.

The backend is responsible for converting a validated **Performance Timeline** into synthesized audio.

This PRD **does not** implement speech synthesis.

Instead, it defines the interface that every backend (Claire, Kokoro, StyleTTS, OpenVoice, etc.) must implement.

This ensures the rest of CSE never depends on a specific model.

# Why this PRD exists

Right now, the Voice Runtime can orchestrate the pipeline, but it has nothing standardized to orchestrate.

This PRD creates that contract.

Later, when we build:

- Claire Voice
- Kokoro backend
- OpenVoice backend
- Future CSE backend

they all simply implement the same interface.

That keeps the runtime completely independent of the synthesis technology.

---

Our pipeline becomes:

```

At that point, the architecture is fully connected.

PRD-006 will be the first step toward actual audio by introducing the streaming layer that a real backend will use.

One small note before you implement: **do not let Antigravity duplicate the backend abstraction you already started in `runtime/voice/backend.py`.** Instead, have it **refactor** that earlier placeholder so the new `acoustic/backend/` package becomes the single source of truth. The `VoiceRuntime` should depend on the new interface, not maintain two parallel backend abstractions. That keeps the codebase clean without changing the overall architecture.
