# Why this PRD is different

Everything before now was **building the engine**.

PRD-008 is **proving the engine**.

If this implementation works, we'll have validated that the architecture can drive a real TTS backend without any architectural changes.

That's a huge milestone because it means our abstractions were correct.

## One final recommendation before you implement

There is one thing I want Antigravity to respect throughout PRD-008:

**Treat Kokoro as a replaceable implementation, never as a dependency of the architecture.**

That means:
- only `backends/kokoro/` should import Kokoro-specific libraries,
- the rest of CSE should remain completely unaware that Kokoro exists,
- if, in the future, you delete `backends/kokoro/` and add `backends/claire/`, the engine should still compile and work without requiring changes to the core.

If we achieve that, then PRD-008 won't just integrate Kokoro—it will prove that the architecture we've built is genuinely backend-agnostic, which has been the goal since the very beginning.
