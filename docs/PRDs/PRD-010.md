## One recommendation before implementation

I want to introduce a project rule starting with PRD-010:

> **Every new feature should include at least one runnable example.**

If a developer can't discover how to use a feature by looking in `examples/`, the feature isn't finished. This will keep the documentation practical, reduce support questions, and make CSE much more approachable for contributors and users alike.
