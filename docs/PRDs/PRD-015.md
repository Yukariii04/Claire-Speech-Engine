# 1. Executive Summary

The Claire Speech Engine has been successfully released on GitHub and PyPI.

This PRD improves usability by introducing a standardized voice discovery, validation, and selection workflow across all supported backends.

The framework architecture shall remain unchanged.

This is the final framework enhancement before Phase 2.5.

---

# 2. Objectives

Implement:

- Backend voice discovery
- Backend voice validation
- Interactive voice selection
- Persistent user configuration
- Improved CLI
- Better runtime errors

Do NOT implement:

- Voice packages
- Voice downloads
- Voice registry
- Community voices
- CPE
- Claire Speech Model

---

# 3. Backend Interface

Every backend SHALL implement:

```python
list_voices()

validate_voice()

load_voice()
```

### `list_voices()`

Return structured metadata instead of plain strings.

Example:

```python
[
    {
        "id": "af_bella",
        "name": "Bella",
        "language": "English (US)",
        "gender": "Female"
    }
]
```

This allows richer CLI output without changing the API later.

---

# 4. Voice Discovery

The CLI command:

```bash
cse voices
```

shall aggregate voices from all registered backends.

Example:

```text
==================================================
KOKORO
==================================================

af_alloy      Alloy        English (US) Female

af_aoede      Aoede        English (US) Female

af_bella      Bella        English (US) Female

...

==================================================
FISH SPEECH
==================================================

default

claire

==================================================
STYLETTS2
==================================================

default
```

No voice list shall be hardcoded in the CLI.

# 9. Configuration

Store user preferences outside the package.

Windows

```
%APPDATA%/ClaireSpeechEngine/config.json
```

Linux

```
~/.config/claire-speech-engine/config.json
```

macOS

```
~/Library/Application Support/ClaireSpeechEngine/config.json
```

Example:

```json
{
  "backend": "kokoro",
  "voice": "af_bella"
}
```

---

# 10. Runtime Behaviour

Current:

```python
engine.load_backend("kokoro")
engine.load_voice("af_bella")
```

New:

```python
engine.load_backend("kokoro")
engine.load_voice()
```

Behaviour:

1. If a voice is explicitly provided → validate and load it.
2. If no voice is provided:
   - Check user configuration.
   - If valid for the current backend, load it.
   - Otherwise load the backend's default voice.

---

# 11. Validation

Every backend SHALL validate ownership.

Example:

```python
engine.load_backend("fishspeech")
engine.load_voice("af_bella")
```

↓

```
Voice "af_bella" is not available for Fish Speech.

Available Fish Speech voices:

• default
• claire

Run:

cse voices
```

Likewise:

```
Voice "claire" belongs to Fish Speech.

Current backend:

Kokoro
```

## One small enhancement I'd recommend

I'd also add one convenience command:

```bash
cse doctor
```

It would print something like:

```text
Claire Speech Engine Diagnostics

✓ CSE Version: 1.0.1-beta
✓ Python Version: 3.12
✓ Backend: Kokoro
✓ Selected Voice: af_bella
✓ Available Voices: 38
✓ Configuration: OK
✓ Dependencies: OK
```

This isn't required for PRD-015, but it's a valuable diagnostic tool for users opening issues on GitHub. They can paste the output into a bug report, making troubleshooting much easier without affecting the core architecture. It's something I'd consider for a future maintenance release rather than this PRD.
