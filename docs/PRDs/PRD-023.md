# 1. Executive Summary

The Performance Graph Builder is responsible for converting a completed Performance Plan into the final immutable Performance Graph.

The Performance Graph is the canonical backend-independent representation consumed by Translator Interfaces.

The Performance Graph Builder performs no communication reasoning, no backend translation, and no acoustic synthesis.

-------------------------------------------------------------------------------

# 2. Background

After the Performance Planning Pass completes, CPE possesses all information required to describe how speech should be delivered.

The Performance Graph Builder assembles this information into the official Performance Graph defined by the Claire Project.

-------------------------------------------------------------------------------

# 3. Objectives

The Performance Graph Builder SHALL:

- Accept a valid Performance Plan.
- Construct one immutable Performance Graph.
- Preserve backend independence.
- Preserve deterministic behavior.
- Produce the canonical representation consumed by translators.

-------------------------------------------------------------------------------

# 4. Scope

## In Scope

- Performance Graph construction.
- Immutable graph creation.
- Validation.
- Backend-independent representation.

## Out of Scope

- Communication reasoning.
- Backend translation.
- Audio generation.
- Acoustic synthesis.
- Runtime optimization.

-------------------------------------------------------------------------------

# 5. Functional Requirements

The Performance Graph Builder SHALL:

- Accept a valid Performance Plan.
- Construct exactly one Performance Graph.
- Preserve all reasoning outputs.
- Reject invalid input.
- Produce immutable output.

-------------------------------------------------------------------------------

# 6. Input

The Performance Graph Builder receives:

- Performance Plan

The Performance Plan SHALL remain unchanged throughout processing.

-------------------------------------------------------------------------------

# 7. Output

The Performance Graph Builder SHALL produce one immutable Performance Graph.

The generated graph SHALL become the sole input accepted by Translator Interfaces.

-------------------------------------------------------------------------------

# 8. Design Principles

The Performance Graph Builder SHALL:

- Perform one responsibility only.
- Never modify the Performance Plan.
- Never perform backend translation.
- Never generate audio.
- Produce deterministic results.
